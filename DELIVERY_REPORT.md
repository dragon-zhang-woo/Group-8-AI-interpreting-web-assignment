## 🔌 API 端点详解

### 1. 翻译接口
```
POST /api/translate
├─ 请求: { text, source_lang, target_lang }
└─ 响应: { success, source_text, translated_text }
```

### 2. 语音转文字
```
POST /api/speech-to-text
├─ 请求: audio file + language
└─ 响应: { success, text, language }
```

### 3. 文字转语音
```
POST /api/text-to-speech
├─ 请求: { text, language }
└─ 响应: audio/wav binary
```

### 4. 评分接口
```
POST /api/score
├─ 请求: { original, user_translation, reference, source_lang, target_lang }
└─ 响应: { pronunciation, fluency, accuracy, total, feedback, suggestions }
```

### 5. 保存记录
```
POST /api/save-record
├─ 请求: { original_text, user_translation, scores }
└─ 响应: { success, message, timestamp }
```

### 6. 查询记录
```
GET /api/records
├─ 请求: (无)
└─ 响应: { success, records[], count }
```

### 7. 获取材料
```
GET /api/material
├─ 请求: (无)
└─ 响应: { success, material }
```

### 8. 健康检查
```
GET /api/health
├─ 请求: (无)
└─ 响应: { status, version, timestamp }
```

## 📈 评分系统工作原理

### 发音标准性评分 (0-3)
```python
检查内容:
1. 文本长度合理性 (+1)
2. 表达规范性检查 (+1)
3. 标点符号使用 (+1)

算法: 内容长度 + 规范检查 + 标点评估
范围: 0-3 分
```

### 语言流畅性评分 (0-3)
```python
检查内容:
1. 句子结构多样性 (+1)
2. 词汇重复频率分析 (+1)
3. 逻辑连接词使用 (+1)

算法: 结构分析 + 词频统计 + 连接词检测
范围: 0-3 分
```

### 翻译准确性评分 (0-3)
```python
相似度计算:
- SequenceMatcher 相似度算法
- 编辑距离计算
- 关键词覆盖率

对标:
相似度 >= 0.8  → 3 分 (优秀)
相似度 >= 0.6  → 2.5 分 (良好)
相似度 >= 0.4  → 2 分 (及格)
相似度 >= 0.2  → 1 分 (欠佳)
相似度 < 0.2   → 0 分 (不及格)
```
